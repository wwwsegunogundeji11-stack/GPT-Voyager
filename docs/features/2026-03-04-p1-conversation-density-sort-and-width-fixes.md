# FEAT-037：会话排序/密度联动虚拟滚动 + 宽度滑杆与按钮溢出修复

## 1. 元信息
- 功能 ID：FEAT-037
- 功能名称：会话排序/密度联动虚拟滚动 + 宽度滑杆与按钮溢出修复
- 日期：2026-03-04
- 作者：Codex
- 分支：N/A

## 2. 概述
针对使用反馈完成一轮可用性与性能并行优化：
- 修复“聊天区宽度滑杆调节后看不到变化”问题。
- 修复会话工作台首屏“导出 HTML”按钮可能超出面板的问题。
- 新增会话索引排序能力（最近活跃 / 标题 A-Z）。
- 新增会话卡片密度切换（标准 / 紧凑），并与虚拟滚动行高联动。

## 3. 需求映射
- 对应 PRD 章节：
  - FR-02 会话索引
  - FR-07 设置
- 覆盖的用户场景：
  - 用户拖动宽度滑杆后页面宽度不变化。
  - 面板初次打开时操作按钮因宽度不足发生溢出。
  - 会话很多时需要更高信息密度和稳定的排序浏览方式。
- 范围说明（In scope / Out of scope）：
  - In scope：
    - 宽度注入策略增强（变量直写 + 更稳健覆盖选择器）
    - 标题行按钮组换行与布局修正
    - 会话排序与密度配置
    - 虚拟滚动按密度动态行高计算
  - Out of scope：
    - 多字段复合排序（如星标优先、备注优先）
    - 用户自定义行高数值

## 4. 设计
- 关键设计决策：
  - 会话列表改为统一数据管线：`筛选 -> 排序 -> 虚拟切片`，避免排序和虚拟滚动状态不一致。
  - 密度切换直接映射虚拟滚动行高估值：
    - 标准：188
    - 紧凑：150
  - 宽度滑杆生效同时覆盖两层策略：
    - 写入 `documentElement/body` CSS 变量
    - 注入样式覆盖 `conversation-turn` 与 `composer` 相关容器
- 取舍说明：
  - 采用固定行高估值，保证实现简单稳定；极端内容长度下误差由 overscan 吸收。

- 主要变更文件：

    - 会话索引能力增强：
      - 新增排序后的 `orderedConversations`。

  - `index.tsx`
    - 标题行与按钮组支持换行，解决窄面板按钮溢出。

    - 新增紧凑密度样式（卡片高度、字体、间距、按钮尺寸）。

## 6. 验证
- 自动化测试：
  - `npm run typecheck`：通过
  - `npm run build`：通过
- 手工验证步骤：
  1. 打开设置中心拖动聊天区宽度滑杆，观察主对话区与输入区宽度实时变化。
  2. 首次展开面板，确认“导出 HTML”等按钮不再溢出到面板外。
  3. 在会话索引切换排序方式，确认列表顺序即时更新且滚动稳定。
  4. 在“标准/紧凑”间切换密度，确认卡片高度变化且滚动性能保持流畅。
  5. 进行筛选后点击“全选当前筛选”，确认与当前排序后的可见集合一致。

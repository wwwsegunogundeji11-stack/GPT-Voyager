# FEAT-036：会话索引虚拟滚动与界面简约化优化

## 1. 元信息
- 功能 ID：FEAT-036
- 功能名称：会话索引虚拟滚动与界面简约化优化
- 日期：2026-03-04
- 作者：Codex
- 分支：N/A

## 2. 概述
按顺序继续开发，补齐“会话很多时性能下降”的核心问题，并同步修正面板观感与对齐问题：
- 会话工作台列表由全量渲染改为虚拟滚动渲染。
- 面板样式继续向白色/米白简约风格收敛，统一卡片、按钮、输入框与筛选区的视觉节奏。

## 3. 需求映射
- 对应 PRD 章节：
  - FR-02 会话索引
  - 非功能需求：性能
- 覆盖的用户场景：
  - 会话索引条目很多时，滚动卡顿、过滤切换变慢。
  - 面板元素对齐不齐、密度和层次不统一，影响日常阅读与操作效率。
- 范围说明（In scope / Out of scope）：
  - In scope：
    - 会话索引虚拟滚动窗口计算（可视区 + overscan + 上下占位）
    - 过滤条件变化后自动回到列表顶部
    - 列表卡片样式对齐与主题细节优化
  - Out of scope：
    - 引入第三方虚拟滚动库
    - 会话卡片密度自定义（紧凑/标准）配置项

## 4. 设计
- 关键设计决策：
  - 使用固定估算行高 + overscan 的轻量虚拟滚动方案，避免新增依赖。
  - 通过 `ResizeObserver + scrollTop` 同步可视区窗口，保证面板尺寸变化后窗口计算仍稳定。
  - 保留既有会话卡片交互（多选、星标、标签、备注、打开会话）不变，仅替换渲染层。
- 取舍说明：
  - 固定行高估算实现成本低、落地快；极端高度差异下可能有轻微偏差，但在当前卡片结构和 overscan 配置下可接受。

## 5. 实现
- 主要变更文件：
  - `src/content/App.tsx`
  - `src/content/index.tsx`
  - `README.md`
  - `PRD.md`
- 核心逻辑：
  - `App.tsx`
    - 新增会话列表可视窗口状态：列表高度、滚动位置。
    - 基于筛选结果计算虚拟窗口：`topSpacerHeight`、`bottomSpacerHeight`、`visibleConversations`。
    - 列表渲染改为“顶部占位 + 可视卡片 + 底部占位”。
    - 在筛选条件变化时自动重置滚动到顶部，避免筛选后落在空白窗口。
  - `index.tsx`
    - 新增 `gv-virtual-spacer` 样式。
    - 调整面板与会话卡片样式：边框、圆角、阴影、间距、控件高度、网格对齐。
    - 收敛为更简约的白/米白视觉语言，降低装饰性背景带来的噪声。
- 数据模型/存储变更：
  - 无新增持久化字段。

## 6. 验证
- 自动化测试：
  - `npm run typecheck`：通过
  - `npm run build`：通过
- 手工验证步骤：
  1. 打开会话工作台，确保会话列表可滚动。
  2. 准备较多会话（滚动 ChatGPT 左栏并重新扫描），观察滚动流畅度。
  3. 切换关键词/文件夹/标签/仅星标筛选，确认列表自动回顶且渲染正常。
  4. 检查会话卡片、按钮、输入框、筛选区是否对齐，整体视觉是否更简洁。

## 7. 风险与回滚
- 已知风险：
  - 固定估算行高策略在极端内容长度下可能出现轻微滚动定位误差。
- 回滚策略：
  - 回退 `App.tsx` 的虚拟窗口计算和占位渲染，恢复原先 `filteredConversations.map` 全量渲染。
  - 回退 `index.tsx` 本次样式块改动。

# FEAT-031：点击页面外侧自动收起侧边栏

## 1. 元信息
- 功能 ID：FEAT-031
- 功能名称：点击页面外侧自动收起侧边栏
- 日期：2026-03-04
- 作者：Codex
- 分支：N/A

## 2. 概述
为提升页面操作连贯性，新增“点击扩展外侧区域自动收起侧边栏”。当侧边栏处于展开状态，用户在 ChatGPT 页面其他区域进行主键点击时，侧边栏会自动收起，减少遮挡。

## 3. 需求映射
- 对应 PRD 章节：
  - FR-01 侧边栏
- 覆盖的用户场景：
  - 用户阅读/编辑聊天内容时希望快速隐藏面板。
- 范围说明（In scope / Out of scope）：
  - In scope：
    - 展开态下点击扩展外侧自动收起
    - 保留面板内交互不受影响
  - Out of scope：
    - 悬停自动展开/收起
    - 收起行为配置开关

## 4. 设计
- 关键设计决策：
  - 使用捕获阶段 `pointerdown` 监听，优先判定外部点击。
  - 仅响应主键点击（`button === 0`），避免右键误触收起。
  - 通过 `composedPath` + `host.contains` 双重判断，兼容 Shadow DOM 事件重定向。
- 取舍说明：
  - 当前不增加设置开关，先保持默认交互简洁一致。

## 5. 实现
- 主要变更文件：
  - `src/content/App.tsx`
  - `README.md`
  - `PRD.md`
- 核心逻辑：
  - `App.tsx` 新增面板展开态外侧点击监听：
    - 若点击路径命中扩展 Host，不收起。
    - 否则执行 `setCollapsed(true)`。

## 6. 验证
- 自动化测试：
  - `npm run typecheck`：通过
  - `npm run build`：通过
- 手工验证步骤：
  1. 展开侧边栏。
  2. 点击聊天页面正文区域，确认侧边栏自动收起。
  3. 点击侧边栏内部按钮/输入框，确认不会自动收起。
  4. 点击右键菜单触发区域，确认不会因右键直接收起。

## 7. 风险与回滚
- 已知风险：
  - 捕获阶段监听可能与个别页面脚本交互时机不同。
- 回滚策略：
  - 回退 `App.tsx` 外侧点击监听 effect。

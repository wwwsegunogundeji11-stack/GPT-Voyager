# FEAT-029：提示词变量预设与共享模板

## 1. 元信息
- 功能 ID：FEAT-029
- 功能名称：提示词变量预设与共享模板
- 日期：2026-03-04
- 作者：Codex
- 分支：N/A

## 2. 概述
按迭代顺序继续推进提示词能力，在原有 `{{变量}}` 填充基础上，新增两项：
1. 变量预设：支持保存当前变量值、快速套用、删除。
2. 共享模板：支持单条提示词导出为 JSON，以及从 JSON 导入模板。

## 3. 需求映射
- 对应 PRD 章节：
  - 8（FR-05 提示词库）
- 覆盖的用户场景：
  - 用户希望同一模板在不同任务下复用“变量组合”。
  - 用户希望在设备/团队之间共享提示词模板。
- 范围说明（In scope / Out of scope）：
  - In scope：
    - 提示词变量预设保存/套用/删除
    - 预设数据随提示词持久化到本地存储与备份
    - 提示词共享模板导入/导出（JSON）
  - Out of scope：
    - 远程云同步
    - 共享模板在线市场

## 4. 设计
- 关键设计决策：
  - 变量预设挂载在 `PromptSnippet` 下，保证备份恢复天然覆盖。
  - 预设去重以“预设名（不区分大小写）”为主，保留最新更新时间。
  - 模板导入采用轻量 JSON 协议，兼容单条与数组两种形态。
- 取舍说明：
  - 暂不引入复杂 schema 迁移系统，优先确保可用与向后兼容。
- 权限/数据/隐私考虑：
  - 仅本地文件读写与本地存储，无新增网络传输。

## 6. 验证
- 自动化测试：
  - `npm run typecheck`：通过
  - `npm run build`：通过
- 手工验证步骤：
  1. 新建含变量的提示词模板，填写变量值并点击“保存预设”。
  2. 清空变量，点击某条预设“套用”，确认变量自动回填。
  3. 删除预设，确认预设列表即时更新。
  4. 在提示词卡片点击“导出模板”，获得 JSON。
  5. 通过“导入共享模板”导入 JSON，确认模板进入提示词库并可使用。
- 测试结果摘要：
  - 预设持久化、模板导入导出、类型检查与构建均通过。

## 7. 风险与回滚
- 已知风险：
  - 第三方手写 JSON 可能存在字段缺失，导入时会被过滤。
- 回滚策略：
  - 回退 `PromptSnippet` 的 `variablePresets` 字段及相关 UI/导入导出逻辑。

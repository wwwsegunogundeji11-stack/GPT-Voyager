# FEAT-034：提示词模板批量导出

## 1. 元信息
- 功能 ID：FEAT-034
- 功能名称：提示词模板批量导出
- 日期：2026-03-04
- 作者：Codex
- 分支：N/A

## 2. 概述
继续推进提示词能力，在单条导出基础上新增“批量导出”能力：用户可在提示词列表勾选多条模板，一次导出为单个 JSON 文件，便于迁移和分享。

## 3. 需求映射
- 对应 PRD 章节：
  - FR-05 提示词库
- 覆盖的用户场景：
  - 用户希望一次导出多个常用模板而不是逐条点击。
- 范围说明（In scope / Out of scope）：
  - In scope：
    - 提示词列表多选
    - 全选筛选结果 / 清空选择
    - 批量导出单文件 JSON
  - Out of scope：
    - 批量删除模板
    - 批量标签编辑

## 4. 设计
- 关键设计决策：
  - 批量导出文件采用 `prompts[]` 数组结构，兼容现有导入解析能力。
  - 导出内容使用已存在的模板转移结构，减少协议分叉。
- 取舍说明：
  - 暂不新增复杂批量操作，先聚焦高频“批量分享/迁移”场景。

## 5. 实现
- 主要变更文件：
  - `src/content/App.tsx`
  - `src/content/index.tsx`
  - `README.md`
  - `PRD.md`
- 核心逻辑：
  - 新增 `selectedPromptIds` 选择状态与有效性清理。
  - 新增 `togglePromptSelection` / `selectAllFilteredPrompts` / `clearPromptSelection`。
  - 新增 `exportSelectedPromptTemplates` 批量导出回调。
  - 提示词卡片头部新增勾选框，顶部新增批量操作按钮。

## 6. 验证
- 自动化测试：
  - `npm run typecheck`：通过
  - `npm run build`：通过
- 手工验证步骤：
  1. 在提示词列表勾选多条模板。
  2. 点击“批量导出”，检查 JSON 文件生成。
  3. 使用“导入共享模板”导入该文件，确认可识别并导入。
  4. 测试“全选筛选结果/清空选择”按钮。

## 7. 风险与回滚
- 已知风险：
  - 导出数量较大时文件体积会变大。
- 回滚策略：
  - 回退 `App.tsx` 的多选与批量导出逻辑及对应 UI。

## 8. 后续事项
- 待办项：
  - 支持批量导出时按标签分组。
  - 支持批量导出格式选择（单文件/多文件）。

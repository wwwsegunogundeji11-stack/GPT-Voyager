# GPT Voyager for ChatGPT 网页版 - 产品需求文档（PRD）

## 1. 文档信息
- 产品名称：GPT Voyager for ChatGPT Web
- 文档版本：v0.1
- 日期：2026-03-04
- 状态：草案
- 参考项目：`https://github.com/Nagi-ovo/gemini-voyager`

## 2. 背景
重度 ChatGPT 用户在网页端需要更强的会话管理、检索与提示词复用能力。原生界面在大规模整理和快速操作方面存在限制。本项目目标是构建一个以本地优先为原则、聚焦效率提升的浏览器扩展。

## 3. 产品目标
- 交付可用的 MVP 扩展，显著提升 ChatGPT 会话管理与检索效率。
- 默认本地存储用户数据，并支持导入导出。
- 建立严格的功能文档机制，保证持续交付质量。

## 4. 成功指标（MVP）
- 首次价值时间：安装后 30 秒内可看到扩展侧边栏。
- 会话操作效率：至少支持一键分类、检索并打开目标会话。
- 数据安全：默认 100% 本地持久化，不强制云端上传。
- 交付纪律：100% 合并功能必须在 `docs/features/` 下有功能文档。

## 5. 目标用户
- 拥有大量 ChatGPT 对话的重度用户。
- 频繁复用提示词的工程师、研究人员、写作者。
- 需要导出 `markdown/json` 做归档的用户。

## 6. 范围
### 6.1 P0（MVP 必做）
- 在 ChatGPT 网页注入侧边栏。
- 读取并展示可见会话列表（标题、URL、时间戳（若可获取））。
- 基于文件夹/标签的会话组织能力。
- 本地索引的关键词检索与过滤。
- 提示词片段库（新增/编辑/删除/复制/插入）。
- 导出当前会话为 Markdown/HTML。
- 本地备份导出/导入（JSON）。
- 内置使用说明页（快速上手、快捷键、常见问题）。
- 基础设置页（行为开关、调试开关等）。

### 6.2 P1（MVP 后）
- 批量操作（多选、批量标签/文件夹移动）。
- 快捷键支持。
- 会话级备注与自定义元数据。
- 会话时间线按节点导出（Markdown/HTML）。
- 会话时间线节点标签与高亮标注（可筛选）。
- 公式收藏与别名管理（收藏、改名、回溯来源）。
- 公式复制支持 LaTeX 与 MathML（Word 粘贴渲染）。
- Mermaid 图表工作台（识别、预览、定位、源码复制）。
- Mermaid 图表收藏与多格式导出（源码/SVG/HTML）。
- 聊天区宽度调节（设置滑杆实时生效）。
- 可选本地向量索引实现语义检索。

### 6.3 P2（未来）
- 可选同步服务（用户可控，默认关闭）。
- 团队共享与协作能力。
- 更深入的使用分析能力。

### 6.4 当前不做
- 强制云账号体系。
- 默认服务端存储。
- 完全替代 ChatGPT 原生界面。

## 7. 用户场景
- 作为用户，我希望通过关键词快速找到并打开目标会话。
- 作为用户，我希望将会话归类到文件夹/标签，便于长期管理。
- 作为用户，我希望不离开聊天页面即可复用提示词模板。
- 作为用户，我希望把重要对话导出为 Markdown/HTML，纳入本地知识库。

## 8. 功能需求
### FR-01 侧边栏
- 仅在 ChatGPT 页面注入。
- 支持收起/展开和宽度持久化。
- 支持点击扩展外侧自动收起侧边栏。

### FR-02 会话索引
- 基于页面可见会话构建本地索引。
- 通过 URL 或会话 ID 去重。
- 会话索引列表在大数据量场景下支持虚拟滚动，保证滚动与筛选响应。
- 支持会话索引排序（按最近活跃 / 按标题）。
- 支持会话卡片密度切换（标准 / 紧凑），并与虚拟滚动联动。

### FR-03 分类管理
- 支持创建文件夹/标签并分配会话。
- 支持按文件夹/标签快速筛选。
- 支持从会话索引中快速打开指定文件夹视图（含未分类）。

### FR-04 搜索
- 基于标题和用户元数据进行本地模糊搜索。
- 常规数据规模下返回速度可接受。

### FR-05 提示词库
- 支持提示词片段的增删改查。
- 支持一键复制，及可选的一键插入输入框。
- 支持变量模板（`{{变量名}}`）填写、预设保存与快速套用。
- 支持提示词共享模板导入/导出（JSON）。
- 支持按勾选项批量导出提示词模板（JSON）。

### FR-06 导出/导入
- 单条会话导出为 Markdown/HTML。
- 支持按时间线筛选结果导出节点（Markdown/HTML）。
- 扩展数据导出/导入为 JSON 备份（包含公式收藏、图表收藏、时间线标注数据）。

### FR-07 设置
- 功能开关与调试模式开关。
- 支持聊天区宽度调节（面板设置实时生效）。
- 基础导入导出控制与版本显示。

### FR-08 使用说明
- 扩展内提供“使用说明”页面，包含快速上手步骤。
- 提供快捷键速查与常见问题，降低首次使用门槛。

### FR-09 图表工作台
- 支持识别会话中的 Mermaid 代码块。
- 支持在侧边栏中预览 Mermaid 图表并定位来源消息。
- 支持 Mermaid 源码复制。
- 支持 Mermaid 图表收藏与别名管理。
- 支持 Mermaid 图表导出（源码/SVG/HTML）。

### FR-10 时间线标注
- 支持给时间线节点添加标签。
- 支持给时间线节点添加高亮标注并同步到页面消息。
- 支持按标签筛选时间线节点。

## 9. 非功能需求
- 性能：常规使用场景（最多约 1000 条索引）下交互流畅。
- 性能补充：会话索引在千级数据量下应避免全量渲染造成的明显掉帧。
- 兼容性：优先 Chromium 系浏览器（Chrome/Edge），再考虑 Firefox。
- 安全性：扩展权限最小化。
- 隐私：默认不向外发送用户数据。
- 可维护性：功能文档严格落盘，模块边界清晰。

## 10. 技术方案（初版）
- 扩展标准：Manifest V3。
- 前端：TypeScript + React（或等价组件化方案）。
- 存储：MVP 使用 `chrome.storage.local`，大数据量可选 IndexedDB。
- 架构模块：
  - `content script`：读取页面状态并注入面板。
  - `background/service worker`：消息路由与特权操作。
  - `ui/panel`：侧边栏应用，承载列表/检索/提示词管理。

## 11. 安全与合规
- 仅申请必要权限（`storage`、`activeTab`、ChatGPT 域名 host 权限）。
- 对删除、批量移除、覆盖导入等危险操作给出明确确认。
- 备份文件仅本地处理，不自动上传。

## 12. 里程碑
- M0：项目脚手架、编码规范、文档流程基线。
- M1：P0 核心能力（侧边栏、列表、检索、分类、提示词库）。
- M2：导入导出与稳定性完善。
- M3：P1 增强与性能优化。

## 13. 风险与缓解
- ChatGPT DOM 结构变动导致选择器失效。
  - 缓解：稳健选择器策略 + `MutationObserver` + 回退机制。
- 扩展权限引发用户顾虑。
  - 缓解：最小权限原则 + 设置项透明化。
- 页面更新导致本地数据不一致。
  - 缓解：显式同步触发与冲突安全合并策略。

## 14. 文档治理（强制）

  - 在 `docs/features/` 下新增一条功能记录：`YYYY-MM-DD-feature-slug.md`。
  - 在 `docs/FEATURE_LOG.md` 增加对应索引记录。
- 任何影响产品范围/行为的变更，必须同步更新 `PRD.md`。

## 15. 完成定义（DoD）
- 代码实现完成。
- 有基础校验/测试证据。
- 在 `docs/features/` 下新增功能文档。
- 已更新 `docs/FEATURE_LOG.md`。
- 若涉及范围调整，已更新 `PRD.md`。

## 16. 待确认问题
- Firefox 支持放在 MVP 还是 M1 之后。
- 提示词插入应模拟键入，还是直接写入输入值。
- 语义检索是否严格本地化，或支持可选外部服务。
